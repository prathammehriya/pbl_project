The Problem

Financial Blindness

Most students and young professionals fail to save not because of low income, but due to **untracked micro-transactions**. Standard banking apps only show history, not insights. Manual spreadsheets are tedious and error-prone.

  • Lack of real-time categorization.
  • No predictive warning system for overspending.
  • Static data without visual context.

The Solution

Core Functionality

Secure Authentication

Implemented using JWT (JSON Web Tokens) and bcrypt hashing. Ensures user data isolation and stateless session management across devices.

Smart Ledger

CRUD operations for incomes and expenses. Features auto-categorization logic and date-filtering to view financial health over specific periods.

Dynamic Visualization

Powered by Chart.js. Converts raw MongoDB data into interactive Doughnut and Bar charts for instant visual comprehension of spending habits.

System Architecture

The application follows a strict Model-View-Controller (MVC) pattern to ensure scalability and code maintainability.

Client Side (View)

React.js sends async Axios requests with JWT headers.

API Layer (Controller)

Express Middleware validates tokens, routes request to specific controllers.

Data Layer (Model)

Mongoose schemas define data structure. MongoDB performs aggregation queries.
